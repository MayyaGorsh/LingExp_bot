# Руководство для исследователя

* Содержание
{:toc}

Это полный гайд по работе с ботом: от первого `/start` до выгрузки
данных. Документ построен так: сначала общая логика интерфейса
исследователя и описание общих настроек эксперимента, затем - по
шаблонам, с форматом CSV, примером и описанием выгрузки.

## Регистрация и роли

После команды `/start` бот предлагает выбрать роль. Исследователь
регистрируется один раз; учётная запись участника создаётся
автоматически при первом переходе по deep-link и не требует
отдельного шага. Один и тот же `telegram_id` может быть и
исследователем, и участником в разных экспериментах - это нормально.

В роли исследователя доступен личный кабинет, в котором можно:

- создавать эксперименты на основе шаблонов или в свободном формате;
- загружать стимулы (CSV) и медиафайлы;
- настраивать параметры (рандомизация, тайм-ауты, демография, листы
  и т. д.);
- активировать эксперимент и получать ссылку для участников;
- просматривать свои эксперименты и их статусы (черновик / активен /
  архив);
- запускать предпросмотр от лица участника;
- скачивать результаты в CSV (с zip-архивом голосовых записей, если
  они есть);
- отправлять текстовые промо-рассылки всем участникам, хотя бы раз
  завершавшим эксперимент в этом боте.

## Жизненный цикл эксперимента

1. **Черновик.** Эксперимент создан, но не виден участникам.
   Допустимо менять любые настройки, перезаливать CSV и медиа.
2. **Активен.** Эксперимент опубликован, виден участникам по
   deep-link. Редактировать стимулы и фазы нельзя: это сломало бы
   консистентность данных у тех, кто уже прошёл. Можно
   деактивировать, удалить или экспортировать данные.
3. **Архив.** Промежуточный статус для приостановленных или
   завершённых экспериментов.

Для активного эксперимента есть «Превью»: исследователь сам проходит
свой эксперимент как участник. Такие сессии помечаются как `preview`
и в итоговую выгрузку не попадают.

Deep-link имеет вид `https://t.me/<имя_бота>?start=exp_<код>`. Бот
оформляет ссылку моноширинным блоком, чтобы клик по ней копировал
адрес, а не открывал - иначе сам исследователь автоматически вошёл
бы в роль участника и испортил бы себе данные.

Удаление эксперимента каскадное: вместе с самим экспериментом
удаляются все его сессии, ответы, метаданные медиа, голосовые блобы
и стимульные блобы. Перед удалением бот предлагает скачать последнюю
выгрузку.

## Структура эксперимента

Любой эксперимент состоит из метаданных (название, приветственное
сообщение, шаблон, статус), одной или нескольких **фаз**, и для
каждой фазы - списка **проб** (trials).

Фаза - это блок проб одного типа: один тип стимула (текст / аудио /
изображение / видео) и один тип ответа. Если в эксперименте сначала
показываются слова с бинарным ответом «слово / не слово», а потом те
же слова оцениваются по Ликерту - это две разные фазы. У большинства
шаблонов фаза одна, у нескольких (Statement Verification, Probe
Recognition, Text Change Detection) - две.

Проба - это одно событие: один стимул и один ответ.

## Общие настройки эксперимента

Эти настройки применяются ко всему эксперименту целиком. Для
free-form-режима часть из них живёт на уровне отдельной фазы.

**Рандомизация порядка стимулов.** Если включена, у каждого
участника пробы внутри фазы перемешиваются. Если выключена - порядок
берётся из CSV.

**Рандомизация позиций кнопок.** На каждой пробе с вариантами ответа
кнопки выкладываются в случайном порядке. Критично для измерения
времени реакции: иначе участник систематически быстрее кликает по
кнопке, к которой ближе палец.

**Перемешивать позиции картинок.** Аналог для шаблонов с двумя или
тремя картинками (Picture Selection, Covered Box): каждому участнику
бот сам тасует позиции изображений в паре или тройке. Корректность
считается по имени файла, поэтому перетасовка ничего не ломает.

**Чистить предыдущие пробы.** По умолчанию включено. Бот удаляет
своё предыдущее сообщение со стимулом перед показом следующей пробы;
участник всегда видит только текущий стимул, не может сравнивать с
прошлыми и не получает эффекта контекста. Если выключено - стимулы
накапливаются в чате. Сообщения участника (текст, голос) удалить
нельзя - это запрет Telegram API.

**Распределение по листам.** Позволяет задать несколько вариантов
эксперимента (лист A, B, C…). Бот сам распределяет участников между
листами поровну; каждый видит только свой лист. Удобно для
разведения критических стимулов и филлеров между группами участников
по схеме Latin square.

**Демография.** Три режима:

- нет - анкета не показывается;
- стандартная - возраст, пол, город, родной язык;
- своя - исследователь загружает CSV со своими вопросами;
  поддерживаются открытые ответы текстом и выбор кнопками.

Ответы на демографию сохраняются в сессии и попадают в итоговую
CSV-выгрузку отдельными колонками.

**Тайм-аут.** Ограничение времени (в секундах) на ответ по каждому
стимулу. Если участник не успел - ставится метка `timed_out`,
эксперимент идёт дальше. Значение «нет» - неограниченное время.

**Перерыв до сброса (idle-таймаут).** Через сколько секунд
бездействия закрывать сессию. Если участник вернулся позже
установленного времени, ему придётся начинать заново. По умолчанию
300 секунд. 0 - отключено, сессия всегда возобновляется с того же
места.

**Тишина в аудио.** Только для шаблонов с аудио-стимулами. Сколько
секунд тишины добавлять в конец каждого аудиофайла. Полезно для
Telegram-плеера, который иногда обрезает окончание.

**Повторное прохождение.** Разрешено или запрещено. Если запрещено,
бот не пустит того же информанта во второй раз - он увидит
сообщение «вы уже проходили этот эксперимент».

**Кастомизация.** Для каждого шаблона можно поменять подписи кнопок
ответа, лейблы делений шкалы Ликерта и тексты инструкций каждой
фазы. Дефолты применяются автоматически, если ничего не задано.

## Как замеряется время реакции

Зависит от типа ответа фазы:

- `buttons`, `likert` - от показа стимула до нажатия кнопки.
  «Чистый» RT.
- `multiple_choice` - от показа стимула до нажатия кнопки «Готово».
  Промежуточные клики по чекбоксам в RT не учитываются.
- `buttons_then_text` - RT фиксируется только на первом шаге
  (кнопка), второй шаг - текстовое обоснование - пишется без RT.
- `open_text`, `voice` - от показа стимула до отправки сообщения,
  то есть RT включает время набора или записи. Telegram не сообщает
  боту, когда пользователь начал печатать, поэтому «чистое» время
  до начала ответа технически измерить нельзя.

## Формат CSV: общие правила

- Разделитель - точка с запятой (`;`). Запятая и табуляция не
  поддерживаются. Бот не пытается угадывать разделитель.
- Кодировка - UTF-8 (предпочтительно) или CP1251. Excel при
  сохранении из «Сохранить как - CSV» обычно даёт нужный формат;
  при сборке файла в LibreOffice следует выбрать «Поле разделено
  точкой с запятой, кодировка UTF-8».
- Первая строка - заголовки колонок. Их имена строго фиксированы
  для каждого шаблона (см. ниже).
- Маркер правильного ответа - символ `*` в начале значения опции
  (`*Слово`, `*Да`). Если правильного ответа нет - маркер не
  ставится.
- Лейблы кнопок берутся прямо из CSV (`opt1`, `opt2`, … значения).
  Они же показываются респонденту. Чтобы поменять надписи, нужно
  изменить значения в CSV или воспользоваться пунктом
  «Кастомизация - лейблы кнопок» в меню настроек.
- Для шаблонов с несколькими листами столбец `list_id` (если
  поддерживается) определяет, к какому листу относится проба;
  если столбца нет, все пробы попадают в лист по умолчанию.

## Загрузка медиафайлов

Для шаблонов с медиа-стимулами (аудио, изображения, видео) после
загрузки CSV исследователь переходит к шагу «Загрузить медиафайлы».
Бот собирает список ожидаемых имён файлов из CSV и принимает их
батчами. Имя присланного файла должно точно совпадать с указанным в
CSV.

Особенности:

- Аудио принимается как файл (`.wav`). Это нужно для корректного
  воспроизведения и точного измерения длительности.
- К аудио-стимулам можно автоматически добавить N секунд тишины в
  конец каждого файла (общая настройка эксперимента «Тишина в
  аудио»). Полезно при обрезке плеера Telegram.
- Картинки хранятся локально (в GridFS-бакете `stimulus_media`),
  потому что Telegram Bot API не отдаёт через `getFile` файлы
  тяжелее 20 МБ - без локальной копии экспорт через несколько
  месяцев мог бы стать недоступным.
- Видео и аудио переотправляются по `file_id` Telegram (там лимиты
  шире), но при первом успешном показе бот кэширует `photo_id` и
  `file_id` для последующих сессий.
- Перед активацией эксперимента бот сам проверяет, что для каждой
  пробы с медиа-стимулом загружен файл с нужным именем. Если
  что-то не хватает - выдаёт список недостающих файлов и не даёт
  опубликовать эксперимент.

## Что попадает в итоговую выгрузку

Для каждой пробы каждого участника в CSV сохраняются:

- `participant_id` - Telegram ID информанта;
- `session_id`, `experiment_id` - идентификаторы сессии и
  эксперимента;
- `template_type` - код шаблона;
- `assigned_list` - имя листа, если эксперимент с распределением;
- `phase_index`, `trial_index` - номера фазы и пробы;
- `stimulus_id` - идентификатор стимула;
- `raw_response` - сырой ответ (текст ответа, имя выбранной кнопки,
  `voice:<id>` для голосовых и т. п.);
- `normalized_response` - нормализованный ответ (приведён к
  каноничной форме, если применимо);
- `is_correct` - `True` / `False` / пусто, если в шаблоне нет
  понятия правильного ответа;
- `reaction_time_ms` - время реакции в миллисекундах;
- `timed_out` - флаг тайм-аута;
- `timestamp` - момент ответа;
- template-specific колонки - зависят от шаблона (например,
  `sentence_id` и `segment` для SPR, `target` и `distractor` для
  Maze, `img_1`, `img_2`, `correct_img` для Picture Selection);
- колонки демографии - отдельная колонка на каждый ключ из анкеты.

Если в эксперименте были голосовые ответы (Sentence Repetition,
Picture Naming с голосом, любые free-form-фазы с типом ответа
«голос»), выгрузка приходит как ZIP-архив: CSV + папка с голосовыми
записями. Имена аудио включают идентификатор пробы - это позволяет
связать запись с конкретной строкой в CSV.

Превью-сессии исследователя в выгрузку не включаются.

# Шаблоны экспериментов

Шаблоны разделены на пять групп: уровень слова, уровень предложения,
аудио, визуальные стимулы и свободный формат. Для каждого шаблона
ниже описано: парадигма и методологический источник, формат CSV,
формат ответа, пример заполнения и какие колонки появятся в
выгрузке сверх базовых.

## Группа 1. Эксперименты на уровне слова

### Lexical Decision

**Парадигма.** Респондент как можно быстрее решает, является ли
предъявленная буквенная последовательность реальным словом языка.

**Методология.** Diependaele et al. (2012).

**CSV.** Колонки:

- `stimulus` - слово или несловарная строка;
- `opt1`, `opt2` - подписи кнопок (по умолчанию «Слово» / «Не
  слово»). Правильный вариант помечается префиксом `*`.

**Пример:**

| stimulus | opt1 | opt2 |
| --- | --- | --- |
| кошка | *Слово | Не слово |
| дерево | *Слово | Не слово |
| нварпт | Слово | *Не слово |
| облуз | Слово | *Не слово |

**Ответ.** Кнопки. Сохраняется выбранная кнопка, корректность, RT.

### Predictability Rating

**Парадигма.** Оценка по шкале Ликерта (по умолчанию 5-балльной),
насколько ожидаемо данное слово в данном контексте.

**Методология.** Gregor de Varda et al. (2024).

**CSV.** Колонки:

- `left_context` - часть предложения до пропуска;
- `target` - целевое слово, ожидаемость которого оценивается;
- `right_context` - часть предложения после пропуска (можно
  оставить пустым).

**Пример:**

| left_context | target | right_context |
| --- | --- | --- |
| Утром она выпила чашку | кофе | и пошла на работу. |
| Он открыл дверь и вошёл в | комнату | где его ждали гости. |
| Студент сдал все экзамены и получил | диплом | с отличием. |

**Ответ.** Шкала Ликерта (число делений и подписи настраиваются в
меню «Кастомизация»). Сохраняется оценка по шкале, RT, исходные
контексты и целевое слово.

### Cloze (Multiple Choice)

**Парадигма.** Закрытое заполнение пропуска. Предложение с пропуском
`___`, ответ - выбор подходящего слова из 2-6 вариантов на кнопках.

**Методология.** Keshavarz & Salimi (2007); Kleijn (2018).

**CSV.** Колонки: `stimulus` (с пропуском `___`), `opt1` … `opt6`
(правильный вариант помечается `*`). Поддерживаются «контекстные»
пробы без пропуска и без правильного ответа - выводятся просто с
кнопкой «Далее» (для разогрева или предъявления контекста).

**Пример:**

| stimulus | opt1 | opt2 | opt3 | opt4 |
| --- | --- | --- | --- | --- |
| Она ___ книгу перед сном каждый вечер. | *читает | пишет | бросает | варит |
| Зимой на улице очень ___. | *холодно | весело | далеко | быстро |
| Утром птицы начинают громко ___. | *петь | плавать | строить | платить |

**Ответ.** Кнопки. Сохраняется выбор, корректность, RT.

### Cloze (Open Ended)

**Парадигма.** То же, но ответ вводится текстом. Введённая строка
сверяется со списком допустимых вариантов из CSV (до 10 на пробу).

**Методология.** Keshavarz & Salimi (2007); Kleijn (2018); Trace
(2020).

**CSV.** Колонки: `stimulus`, `correct1` … `correct10`.

**Пример:**

| stimulus | correct1 | correct2 | correct3 |
| --- | --- | --- | --- |
| Она ___ книгу перед сном каждый вечер. | читает | читала | читала книгу |
| Зимой на улице очень ___. | холодно | морозно | студёно |
| Утром птицы начинают громко ___. | петь | щебетать | чирикать |

**Ответ.** Открытый текст. Сохраняется введённая строка и пометка,
попал ли ответ в список допустимых.

### Word Translation (Closed)

**Парадигма.** Слово на исходном языке, выбор перевода из 2-6
кнопок.

**Методология.** Golubović & Gooskens (2015).

**CSV.** Колонки: `stimulus`, `opt1` … `opt6` (правильный - `*`).

**Пример:**

| stimulus | opt1 | opt2 | opt3 | opt4 |
| --- | --- | --- | --- | --- |
| будинок | *дом | магазин | город | река |
| кішка | *кошка | собака | птица | рыба |
| книжка | *книга | тетрадь | журнал | газета |

**Ответ.** Кнопки. Сохраняется выбор, корректность, RT.

### Word Translation (Open)

**Парадигма.** Перевод вводится текстом. RT не фиксируется
(Telegram не отслеживает момент начала ввода).

**Методология.** Golubović & Gooskens (2015).

**CSV.** Колонки: `stimulus`, `correct1` … `correct6`.

**Пример:**

| stimulus | correct1 | correct2 | correct3 |
| --- | --- | --- | --- |
| будинок | дом | домик |  |
| кішка | кошка | кот | котик |
| книжка | книга | книжка |  |

**Ответ.** Открытый текст. Сохраняется введённая строка и пометка
попадания в список допустимых переводов.

## Группа 2. Эксперименты на уровне предложения

### Sensicality Judgment

**Парадигма.** Бинарная оценка осмысленности предложения, на
скорость.

**Методология.** Bambini et al. (2013).

**CSV.** Колонки: `stimulus`, `opt1`, `opt2`. Маркер `*` не
ставится - «правильного» ответа в этом шаблоне нет.

**Пример:**

| stimulus | opt1 | opt2 |
| --- | --- | --- |
| Девочка пила молоко из стакана. | Осмысленно | Неосмысленно |
| Камень мечтал о летних каникулах. | Осмысленно | Неосмысленно |
| Зелёный ветер читал утреннюю газету. | Осмысленно | Неосмысленно |

**Ответ.** Кнопки. Сохраняется выбор и RT.

### Acceptability Judgment (AJT)

**Парадигма.** Оценка естественности предложения для носителя.
Самый гибкий шаблон. Поддерживает:

- разный формат ответа: если в CSV заданы 2-10 кнопок - это
  bipolar / yes-no / несколько градаций; если опций нет - числовая
  шкала Ликерта из настроек эксперимента;
- три режима подачи: `single` (одиночная подача, по одному
  предложению на пробу), `joint_one_rating` (совместная подача двух
  предложений, одна общая оценка пары - например, «1 лучше» / «2
  лучше»), `joint_two_ratings` (совместная подача двух предложений,
  каждое оценивается отдельно).

**Методология.** Chomsky (1957); Marty et al. (2020).

**CSV.** Колонки: `stimulus`, опционально `stimulus2` (для
joint-режимов), `opt1` … `opt10`.

**Пример (одиночная подача, 5-балльная шкала):**

| stimulus | opt1 | opt2 | opt3 | opt4 | opt5 |
| --- | --- | --- | --- | --- | --- |
| Мальчик съел яблоко в саду. | Совсем неприемлемо | Скорее неприемлемо | Затрудняюсь | Скорее приемлемо | Полностью приемлемо |
| Прохожих на собака лаяла громко. | Совсем неприемлемо | Скорее неприемлемо | Затрудняюсь | Скорее приемлемо | Полностью приемлемо |

**Пример (joint, одна оценка на пару):**

| stimulus | stimulus2 | opt1 | opt2 |
| --- | --- | --- | --- |
| Мальчик съел яблоко в саду. | Мальчик яблоко съел саду в. | *1 лучше | 2 лучше |
| Собака громко лаяла на прохожих. | Прохожих на собака лаяла громко. | 1 лучше | *2 лучше |

**Ответ.** Кнопки или шкала Ликерта (зависит от CSV и настроек). В
выгрузку идут оба стимула, оценки и RT.

### Truth Value Judgment Task (TVJT)

**Парадигма.** Респонденту даётся (опционально) короткая история и
утверждение о ней; задача - оценить истинность утверждения и
(опционально) обосновать свой выбор текстом.

**Методология.** Lidz & Musolino (2002).

**CSV.** Колонки: `context` (опционально), `stimulus`, `opt1` …
`opt6` (правильный - `*`), `ask_justification` (`да` / `нет` -
запрашивать ли обоснование; можно включать только для критических
проб, для филлеров - нет).

**Пример:**

| context | stimulus | opt1 | opt2 | ask_justification |
| --- | --- | --- | --- | --- |
| Лошадь принесла четырёх котят в корзинку, а одного оставила во дворе. | Лошадь принесла всех котят в корзинку. | Правильно | *Неправильно | да |
| Каждый из трёх мальчиков поймал по одной рыбе. | Каждый мальчик поймал рыбу. | *Правильно | Неправильно | нет |
| Из пяти учеников задачу решили только двое. | Все ученики решили задачу. | Правильно | *Неправильно | да |

**Ответ.** Кнопки + (если включён `ask_justification`) текст.
Сохраняются: выбор, RT первого клика, текст обоснования.

### Statement Verification

**Парадигма.** Респондент оценивает истинность утверждений
(например, «У слона четыре ноги»), затем проходит контрольную фазу
с вопросами на знание фактов. Позволяет отделить эффект иллюзии
от незнания.

**Методология.** Bredart & Modolo (1988).

**CSV.** Две фазы, два файла.

*Фаза 1 - оценка утверждений:* колонки `stimulus`, `opt1` … `opt6`
(правильный - `*`).

| stimulus | opt1 | opt2 | opt3 |
| --- | --- | --- | --- |
| Моисей взял на ковчег каждой твари по паре. | *Верно | Неверно | Не знаю |
| Столица Австралии - Сидней. | Верно | *Неверно | Не знаю |
| Земля вращается вокруг Солнца. | *Верно | Неверно | Не знаю |

*Фаза 2 - контрольные вопросы:* колонки `question`, `opt1` …
`opt6`. Число опций у разных вопросов может отличаться (пустые
ячейки в строке - это просто отсутствие соответствующих кнопок).

| question | opt1 | opt2 | opt3 | opt4 |
| --- | --- | --- | --- | --- |
| Кто на самом деле взял животных на ковчег? | Моисей | *Ной | Авраам | Иисус |
| Какой город является столицей Австралии? | Сидней | Мельбурн | *Канберра | Перт |
| Кто написал «Войну и мир»? | Достоевский | *Толстой | Тургенев |  |

**Ответ.** Кнопки. Сохраняются ответы обеих фаз и RT.

### Self-Paced Reading (SPR)

**Парадигма.** Предложение предъявляется по сегментам (слово или
группа слов). Каждое нажатие «Далее» добавляет следующий фрагмент.
Время до нажатия фиксируется как время чтения сегмента.

**Методология.** Swets et al. (2007); Marsden et al. (2018).

**CSV.** Колонки: `sentence_id` (одинаковый для всех сегментов
одного предложения), `segment` (сам сегмент). Один логический
стимул разворачивается в несколько строк CSV.

**Пример:**

| sentence_id | segment |
| --- | --- |
| 1 | Пока |
| 1 | охотник |
| 1 | стрелял |
| 1 | олень |
| 1 | перепрыгнул |
| 1 | через |
| 1 | забор. |
| 2 | Пока |
| 2 | охотник |
| 2 | стрелял |
| 2 | в |
| 2 | оленя |
| 2 | птица |
| 2 | улетела |
| 2 | далеко. |

**Ответ.** Одна кнопка «Далее» на каждом сегменте. Сохраняются:
`sentence_id`, `segment`, время чтения каждого сегмента.

### Maze Task

**Парадигма.** Пословное чтение, на каждой позиции - два варианта
(целевое слово и дистрактор), участник выбирает грамматически
корректное продолжение. При ошибке оставшаяся часть предложения
пропускается, бот переходит к следующему.

**Методология.** Boyce et al. (2020).

**CSV.** Колонки: `target` (правильное слово), `distractor`
(неправильный вариант или `$$$` для первого слова предложения,
которое выводится без выбора). Позиции кнопок бот тасует
автоматически попробно.

**Пример:**

| target | distractor |
| --- | --- |
| Кошка | $$$ |
| сидела | вчерашний |
| на | от |
| высоком | круглая |
| заборе | молнией |
| и | но |
| смотрела | чертила |
| на | от |
| птиц. | домов. |

**Ответ.** Кнопки. Сохраняются: `target`, `distractor`, выбор, RT.

### Text Change Detection

**Парадигма.** Один и тот же короткий текст предъявляется дважды.
Сначала участник читает оригинал и нажимает «Далее» (фиксируется
время чтения); затем то же сообщение редактируется в повторное
предъявление с двумя кнопками («было / не было изменения»). При
ответе «было» бот дополнительно спрашивает, какое слово было в
оригинале и на какое заменили - корректность считается только при
точной идентификации обоих слов.

**Методология.** Sanford et al. (2005).

**CSV.** Колонки: `text_original`, `text_repeated`,
`changed_word_original`, `changed_word_new`, `opt1` (= «было»),
`opt2` (= «не было»). Если изменений нет, поля
`changed_word_original` и `changed_word_new` оставляются пустыми.

**Пример:**

| text_original | text_repeated | changed_word_original | changed_word_new | opt1 | opt2 |
| --- | --- | --- | --- | --- | --- |
| Мужчина, который навестил сестру, привёз цветы из магазина. | Мужчина, который навестил сестру, привёз розы из магазина. | цветы | розы | *Изменение было | Изменения не было |
| Девочка, которая читала книгу, сидела на скамейке в парке. | Девочка, которая читала книгу, сидела на скамейке в парке. |  |  | Изменение было | *Изменения не было |
| Повар, который готовил обед, добавил специи в бульон на кухне. | Повар, который готовил обед, добавил соль в бульон на кухне. | специи | соль | *Изменение было | Изменения не было |

**Ответ.** Двухступенчатый - кнопка, затем (если «было») два
текстовых ответа. В выгрузке: оригинал, повторное предъявление,
оригинальное и новое слово, время чтения, ответы пользователя.

### Probe Recognition

**Парадигма.** Двухфазный эксперимент. На фазе запоминания
респондент последовательно читает и запоминает фразы (без выбора,
переход по «Далее»); на фазе тестирования предъявляются тестовые
фразы с одним словом, написанным заглавными буквами, и участник
решает, встречалось ли выделенное слово на фазе запоминания.

**Методология.** Light & Carter-Sobell (1970); Klein & Murphy
(2001).

**CSV.** Две фазы, два файла.

*Фаза 1 - запоминание:* колонка `stimulus`.

| stimulus |
| --- |
| клубничное варенье |
| горячий шоколад |
| тенистая улица |
| свежая газета |

*Фаза 2 - тестирование:* колонки `stimulus`, `opt1` … `opt6`
(обычно «Да» / «Нет», правильный - `*`). Слово, которое
проверяется на узнавание, выделено заглавными буквами.

| stimulus | opt1 | opt2 |
| --- | --- | --- |
| клубничное ВАРЕНЬЕ | *Да | Нет |
| горячий ШОКОЛАД | *Да | Нет |
| автомобильная ПРОБКА | Да | *Нет |
| тёмный ШОКОЛАД | Да | *Нет |

**Ответ.** На фазе 1 - «Далее», на фазе 2 - кнопки. Сохраняется
выбор, корректность, RT.

### Interpretation Generation

**Парадигма.** Респондент читает предложение, нажимает «Далее»;
стимул исчезает, и участник вводит парафраз или интерпретацию
текстом.

**Методология.** Patson et al. (2009).

**CSV.** Колонка `stimulus`.

**Пример:**

| stimulus |
| --- |
| Пока студент читал, книга упала со стола. |
| После того как солдаты пообедали, лошади были готовы к маршу. |
| Когда Маша постирала, бельё высохло за два часа. |
| Девочка сказала бабушке, что она устала. |

**Ответ.** Открытый текст. Сохраняется парафраз.

## Группа 3. Эксперименты с аудиостимулами

### Forced Choice Identification

**Парадигма.** На каждом шаге респондент слушает аудио-стимул и
выбирает один из заданных классов на кнопках (2-6 альтернатив).
Подходит для задач фонемной категоризации, идентификации эмоций,
оценки акцента и т. д.

**Методология.** Kapnoula et al. (2017).

**CSV.** Колонки: `audio_filename`, `opt1` … `opt6`, `repeats`
(опционально - сколько раз повторить стимул в эксперименте; по
умолчанию 1). Если у задачи есть «правильный» класс - помечается
`*`.

**Пример:**

| audio_filename | opt1 | opt2 | repeats |
| --- | --- | --- | --- |
| work1.wav | *work | walk | 3 |
| work2.wav | *work | walk | 3 |
| walk1.wav | work | *walk | 3 |
| walk2.wav | work | *walk | 3 |

Аудиофайлы загружаются отдельным шагом (см. раздел «Загрузка
медиафайлов»).

**Ответ.** Кнопки. Сохраняется выбор, RT, имя файла, число
повторов.

### Sentence Repetition

**Парадигма.** Респондент слушает аудио-стимул и записывает
голосовое сообщение с устным повторением. Разметка корректности -
на стороне исследователя, постфактум.

**Методология.** Potter & Lombardi (1998).

**CSV.** Колонка `audio_filename`.

**Пример:**

| audio_filename |
| --- |
| type1_separate_01.wav |
| type2_prime_second_clause_02.wav |
| type3a_prime_first_03.wav |
| type3b_target_first_04.wav |
| filler_01.wav |
| filler_02.wav |

**Ответ.** Голосовое сообщение. Аудиофайлы ответов сохраняются в
GridFS (бакет `voice_answers`) и прикладываются к экспорту в zip.

## Группа 4. Эксперименты с визуальными стимулами

### Picture Selection

**Парадигма.** Предъявляется предложение и пара изображений;
участник выбирает картинку, которая лучше соответствует описанию.

**Методология.** Qiu et al. (2025).

**CSV.** Колонки: `stimulus`, `pair_id`, `img_1_filename`,
`img_2_filename`, `correct_img` (опционально - имя файла
правильной картинки). Позиции картинок можно перемешивать
автоматически (общая настройка «Перемешивать позиции картинок»).
Корректность считается по имени файла, поэтому рандомизация ничего
не ломает.

**Пример:**

| stimulus | pair_id | img_1_filename | img_2_filename | correct_img |
| --- | --- | --- | --- | --- |
| На картинке птица. | птица_01 | robin_photo.jpg | iguana_photo.jpg | robin_photo.jpg |
| На картинке ящерица. | ящерица_02 | iguana_photo.jpg | macaw_photo.jpg | iguana_photo.jpg |
| На картинке морское животное. | море_03 | whale_photo.jpg | deer_photo.jpg | whale_photo.jpg |

**Ответ.** Кнопки «1» / «2». Сохраняются: пара, оба имени файла,
имя правильной картинки, выбор, корректность, RT.

### Covered Box

**Парадигма.** Расширение Picture Selection. К двум видимым
картинкам добавляется третья - «закрытая коробка», которую участник
выбирает, если ни одна видимая картинка не подходит. Используется
для разведения семантики и прагматических выводов.

**Методология.** Huang et al. (2013).

**CSV.** Колонки: `stimulus`, `pair_id`, `img_1_filename`,
`img_2_filename`, `img_3_filename`, `correct_img` (опционально).
По соглашению третья позиция - это `covered.png` (любая картинка,
изображающая закрытый ящик).

**Пример:**

| stimulus | pair_id | img_1_filename | img_2_filename | img_3_filename | correct_img |
| --- | --- | --- | --- | --- | --- |
| Покажи оленя. | олень_01 | deer_photo.jpg | whale_photo.jpg | covered.png | deer_photo.jpg |
| Покажи птицу. | птица_02 | whale_photo.jpg | clownfish_photo.jpg | covered.png | covered.png |
| Покажи рептилию. | рептилия_03 | iguana_photo.jpg | clownfish_photo.jpg | covered.png | iguana_photo.jpg |

**Ответ.** Кнопки «1» / «2» / «3». Сохраняются те же поля, что и в
Picture Selection, плюс третья картинка.

### Picture Naming

**Парадигма.** Респонденту предъявляется изображение, нужно назвать
его одним словом или короткой именной группой. RT не фиксируется
(Telegram не отслеживает момент начала ввода).

**Методология.** Krautz & Keuleers (2022).

**CSV.** Колонки: `img_filename`, `correct1` … `correct4`
(опционально - допустимые варианты названия).

**Пример:**

| img_filename | correct1 | correct2 |
| --- | --- | --- |
| deer_photo.jpg | олень |  |
| iguana_photo.jpg | игуана | ящерица |
| macaw_photo.jpg | ара | попугай |
| robin_photo.jpg | зарянка | малиновка |
| whale_photo.jpg | кит |  |

**Ответ.** Открытый текст или голосовое сообщение (выбирается в
настройках фазы). Сохраняется ответ и пометка попадания в список
допустимых вариантов.

### Video Task

**Парадигма.** На каждой пробе предъявляется видео и набор кнопок,
заданных исследователем (например, градации шкалы Ликерта или
варианты act-out-задачи).

**Методология.** Deliens et al. (2018).

**CSV.** Колонки: `video_filename`, `opt1` … `opt7`. У всех проб
должно быть одинаковое число вариантов.

**Пример:**

| video_filename | opt1 | opt2 | opt3 | opt4 | opt5 | opt6 | opt7 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| bunny_video.mp4 | 1 - Совсем не милое | 2 | 3 | 4 - Нейтрально | 5 | 6 | 7 - Очень милое |
| dog_video.mp4 | 1 - Совсем не милое | 2 | 3 | 4 - Нейтрально | 5 | 6 | 7 - Очень милое |
| fox_video.mp4 | 1 - Совсем не милое | 2 | 3 | 4 - Нейтрально | 5 | 6 | 7 - Очень милое |
| wolf_video.mp4 | 1 - Совсем не милое | 2 | 3 | 4 - Нейтрально | 5 | 6 | 7 - Очень милое |

**Ответ.** Кнопки. Сохраняется выбор и RT.

## Группа 5. Свободный формат (free-form)

Конструктор, в котором исследователь сам собирает последовательность
фаз, не привязываясь ни к одному из стандартных дизайнов. Подходит,
если нужен нестандартный сценарий или нужно комбинировать несколько
разнотипных блоков в одном прохождении.

Каждая фаза собирается из трёх независимых выборов:

1. **Тип стимула:** текст / аудио / изображение / видео.
2. **Тип ответа:** кнопки / шкала Ликерта / множественный выбор
   (несколько чекбоксов и кнопка «Готово») / кнопки + текст
   (бинарный выбор и следом текстовое обоснование) / открытый текст
   / голосовое сообщение.
3. **Инструкция фазы, CSV со стимулами и per-phase настройки:**
   рандомизация порядка, тайм-аут, рандомизация позиций кнопок и
   картинок.

Для каждой комбинации (тип стимула × тип ответа) бот генерирует на
лету CSV-пример заполнения с подписями колонок: `stimulus`,
`caption` (для медиа), варианты ответа или деления шкалы,
`follow_up_prompt` (для «кнопок + текст»). Правильные варианты
помечаются `*`.

Free-form позволяет, например:

- показать видео и собрать комментарии текстом;
- дать прослушать пару аудио и попросить выбрать «более
  естественное» из нескольких;
- собрать рейтинг изображений по своим критериям;
- провести многошаговый опросник с разнотипными вопросами.

# Промо-рассылки

В главном меню исследователя есть пункт «Рассылка участникам». Бот
показывает, сколько уникальных пользователей за всё время хотя бы
раз завершили эксперимент в этом боте (по всем экспериментам всех
исследователей). Исследователь вводит текст рассылки, подтверждает
отправку - и бот доставляет сообщение всем этим пользователям.
Используется для приглашения на новые волны экспериментов.

Рассылка - премиум-функция. Респонденты могут отказаться от
получения рассылок по команде `/unsubscribe`; после этого они
больше не получат ни одной рассылки ни от одного исследователя,
пока сами не подпишутся обратно.

# Чек-лист перед запуском

Прежде чем приглашать информантов, имеет смысл пройтись по
короткому списку:

- стимулы соответствуют гипотезе, дистракторы сбалансированы;
- CSV прочитался без ошибок (бот покажет ошибки сразу при загрузке);
- все медиафайлы загружены, имена в CSV точно совпадают с
  присланными;
- инструкции фаз понятны без сопровождения автора - имеет смысл
  прочитать их глазами участника;
- проведено «Превью» (преимущественно с двух разных аккаунтов и
