# Telegram-бот для проведения психо- и нейролингвистических экспериментов

**Автор:** Майя Горшенина.
**ВКР**, НИУ ВШЭ, ОП «Фундаментальная и компьютерная лингвистика», 2026.

## Цель работы

Целью нашей работы является разработка универсального
Telegram-бота, в котором лингвист-исследователь может собрать
экспериментальную сессию из готовых шаблонов основных
психо- и нейролингвистических парадигм, загрузить стимулы и
медиафайлы, провести эксперимент с информантами без установки
специализированного программного обеспечения на их стороне и
выгрузить полученные данные в формате CSV.

Бот покрывает парадигмы по обработке слова (lexical decision,
predictability rating, cloze, word translation), по обработке
предложения (sensicality judgment, acceptability judgment, truth
value judgment, statement verification, self-paced reading, maze,
text change detection, probe recognition, interpretation
generation), по аудиальному восприятию (forced choice
identification, sentence repetition) и по работе с визуальными
стимулами (picture selection, covered box, picture naming, video
task), а также допускает сборку нестандартных дизайнов в режиме
free-form.

## Документация

Документация разделена на два руководства:

- [Руководство для исследователя](researchers.md) - как
  зарегистрироваться, создать эксперимент, загрузить стимулы и
  медиафайлы, опубликовать ссылку и забрать данные. Для каждого
  шаблона приведены формат CSV, пример заполнения и описание того,
  что попадает в выгрузку.
- [Руководство для разработчика](developers.md) - карта
  репозитория, описание каждого модуля и его публичных функций,
  правила добавления нового шаблона эксперимента, схема MongoDB,
  условия деплоя.

## Возможности

- 17 шаблонов стандартных парадигм и конструктор для нестандартных
  сценариев;
- демография (стандартная анкета, своя CSV-анкета, отказ);
- распределение участников по листам (A, B, C…), рандомизация
  порядка стимулов и позиций кнопок и картинок;
- ограничение времени ответа, тайм-аут сессии, контроль повторного
  прохождения;
- кастомизация инструкций фаз, подписей кнопок ответа, шкалы
  Ликерта;
- предпросмотр эксперимента для исследователя без попадания в
  итоговые данные;
- экспорт в CSV и в zip-архив с голосовыми записями;
- промо-рассылка прошлым информантам.

## Технологии

Python 3, aiogram (FSM, роутеры, middlewares), MongoDB через
Motor, GridFS для бинарных данных (голосовые ответы,
картинки-стимулы). Развёртывание - локально (Windows) или на
удалённом сервере; конфигурация через `.env` (токен бота, строка
подключения к MongoDB, имя базы).
